$p.data

5.0_5.5005A.20230904.185702

DataColletion을 제어하는 기능을 제공합니다.
제공하는 기능으로는 복수의 데이터 객체에서 데이터 꺼내오기, 데이터 할당하기, 데이터 객체 생성하기 등이 있습니다.

Property Detail

Event Detail

Method Detail

create(dataCollectionInfo)

DataList 또는 DataMap을 생성합니다. 첫 번째 인자(dataCollectionInfo)에 생성할 객체의 구조를 JSON 객체 또는 XML 형식의 문자열로 정의합니다.

Parameter
  • Name
    dataCollectionInfo
  • Type
    JSON||String
  • Required
    Y
  • Description

    DataList 또는 DataMap의 구조를 정의한 JSON 객체 또는 XML 문자열.

Sample

// 예시 1) dataList 생성 (JSON 객체) var jsnOptionDataList = { "id": "dlt_exam_json", "type": "dataList", "option": { "baseNode": "list", "repeatNode": "map" }, "columnInfo": [ { "id": "code", "name": "코드", "dataType": "text" }, { "id": "label", "name": "코드명", "dataType": "text" } ] }; $p.data.create(jsnOptionDataList); // 예시 2) dataMap 생성 (JSON 객체) var jsnOptionDataMap = { "id": "dma_exam_json", "type": "dataMap", "option": { "baseNode": "map" }, "keyInfo": [ { "id": "code", "name": "코드", "dataType": "text" }, { "id": "label", "name": "코드명", "dataType": "text" } ] }; $p.data.create(jsnOptionDataMap); // 예시 3) dataList 생성 (XML 문자열) var strDataList = '<w2:dataList id="dlt_exam_xml" baseNode="vector" repeatNode="item" valueNode=""> ' + '<w2:columnInfo>' + '<w2:column id="code" name="코드" dataType="text"></w2:column>' + '<w2:column id="label" name="코드명" dataType="text"></w2:column>' + '</w2:columnInfo>' + '</w2:dataList>'; $p.data.create(strDataList); // 예시 4) dataMap 생성 (XML 문자열) var strDataMap = '<w2:dataMap baseNode="map" id="dma_exam_xml">' + '<w2:keyInfo>' + '<w2:key id="code" name="코드" dataType="text"></w2:key>' + '<w2:key id="label" name="코드명" dataType="text"></w2:key>' + '</w2:keyInfo>' + '</w2:dataMap>'; $p.data.create(strDataMap);

getAllDataCollection(type)

실행된 모든 화면의 DataCollection 정보를 반환합니다.
첫 번째 인자(type)에 지정한 값에 따라 DataColletion의 실행 시점의 ID만 반환받거나 DataColletion 객체를 반환받을 수 있습니다.
이 함수를 호출한 페이지와 동일한 Window 객체에 생성된 DataCollection이 대상입니다.
IFrame의 소스 페이지 또는 '$p.openPopup'으로 생성한 팝업의 유형이 'iframePopup' 또는 'browserPopup'인 경우에는 Window 객체가 다르기 때문에 제외됩니다.
이 함수는 'Page Scope'이 적용되지 않습니다.

Parameter
  • Name
    type
  • Type
    string
  • Required
    N
  • Description

    [object] DataCollection 정보 유형. 'object'로 지정된 경우 객체를 반환하고 그 외의 경우에는 ID 문자열을 반환합니다.

Return
  • Type
    Array
  • Description
    DataCollection의 ID 문자열 또는 객체를 담은 배열.

Sample

// 예시 1) 이 함수를 호출한 화면의 Window 객체에 속한 모든 DataColletion의 ID를 반환 // 실행 화면에 WFrame 컴포넌트가 구성되어 있고 ID가 'wfm_exam1'인 경우, // 실행 화면에 정의된 "dataList1"와 WFrame 소스 화면에 정의된 "dataList1_w"와 "dataMap1_w"의 실행 시점의 ID가 반환된 예시입니다. var arrResult = $p.data.getAllDataCollection(); // return 예시) ["mf_dataList1","mf_wfm_exam1_dataList1_w","mf_wfm_exam1_dataMap1_w"] // 예시 2) 이 함수를 호출한 화면의 Window 객체에 속한 모든 DataColletion의 객체를 반환. var arrResult = arrResult = $p.data.getAllDataCollection("object"); // return 예시) // [ // {id: 'mf_dataList1', element: {…}, …}, // {id: 'mf_wfm_exam1_dataList1_w', element: {…}, …}, // {id: 'mf_wfm_exam1_dataMap1_w', element: {…}, …} // ] // 반환된 객체는 Data 객체가 가진 함수를 사용할 수 있습니다. // 첫 번째 Data 객체의 유형을 출력하고자 할 때 var strType = arrResult[0].getObjectType(); // return 예시) 'dataList'
getBroadcast()

전체 DataCollection의 Broadcast 기능의 활성화 여부를 반환합니다.
(제약 사항)
이 함수를 호출한 페이지와 동일한 Window 객체에 생성된 DataCollection이 대상입니다.
IFrame의 소스 페이지 또는 '$p.openPopup'으로 생성한 팝업의 유형이 'iframePopup' 또는 'browserPopup'인 경우에는 Window 객체가 다르기 때문에 제외됩니다.
이 함수는 'Page Scope'이 적용되지 않습니다.

Return
  • Type
    boolean
  • Description
    [true, false] 전체 DataCollection의 Broadcast 기능의 활성화 여부.

Sample

// 예시) 전체 DataCollection의 Broadcast 기능의 활성화 여부 반환. var blnResult = $p.data.getBroadcast(); // return 예시 - 활성화된 경우) true // return 예시 - 비활성화된 경우) false

remove(dataCollectionID)

DataList, DataMap, LinkedDataList, AliasDataList, AliasDataMap를 삭제합니다.
LinkedDataList와 연결된 DataList가 삭제되면, LinkedDataList도 함께 삭제됩니다.
그러나 AliasDataList, AliasDataMap에 연결된 DataList 또는 DataMap이 삭제되면, AliasDataList와 AliasDataMap은 삭제되지 않고 undefined가 할당됩니다.

Parameter
  • Name
    dataCollectionID
  • Type
    String
  • Required
    Y
  • Description

    DataList, DataMap, LinkedDataList, AliasDataList, AliasDataMap의 ID.

Sample

// 예시) 화면에 정의된 id가 'dataMap1'인 데이터 객체를 삭제합니다. $p.data.remove("dataMap1");
set(dataType, dataObject, :N>)

여러 개의 DataList 또는 DataMap에 데이터를 할당합니다.
(LinkedDataList, AliasDataList, AliasDataMap은 제외됩니다. 이 객체들의 데이터는 참조한 DataList 또는 DataMap에 할당해야 합니다.)

  • Type

  • Required
    Y
  • Description

    [ARRAY, JSON, XML] 데이터의 타입. 두 번째 인자(dataObject)에 지정할 데이터의 타입을 지정합니다.

  • Name
    dataObject
  • Type
    JSON||ARRAY||XML
  • Required
    Y
  • Description

Sample

// (DataCollection의 소스 예시) <w2:dataList id="dataList1" baseNode="list" repeatNode="map" saveRemovedData="true"> <w2:columnInfo> <w2:column id="code" name="코드" dataType="text"></w2:column> <w2:column id="label" name="코드명" dataType="text"></w2:column> </w2:columnInfo> </w2:dataList> <w2:dataMap id="dataMap1" baseNode="map"> <w2:keyInfo> <w2:key id="lang" name="언어코드" dataType="text"></w2:key> <w2:key id="output" name="출력값" dataType="text"></w2:key> </w2:keyInfo> </w2:dataMap> // 예시 1) 첫 번째 인자(dataType)별 // 예시 1-1) JSON 유형 // 두 번째 인자 값의 데이터는 JSON 유형으로 정의되어 있으며, 총 3개의 KEY("dataList1", "dataMap1", "undefined_data")로 구성되어 있습니다. $p.data.set( "JSON", { "dataList1": [ { "code": "C0001", "label": "CODE1" } ], "dataMap1": { "lang": "en", "output": "WebSquare" }, "undefined_data": { "code": "C0001", "label": "CODE1" } } ); // 예시 1-2) ARRAY 유형 - 실행 결과 // KEY가 "dataList1"와 "dataMap1"은 화면에 정의된 데이터 객체와 ID가 동일하기 때문에 값이 할당됩니다. // 반면, "undefined_data"는 화면에 정의된 데이터 객체의 ID가 일치하는 객체가 없기 때문에 무시됩니다. // 예시 1-3) XML 유형 let strXML = '' + '<map>' + ' <list id="dataList1">' + ' <map>' + ' <code>C0001</code>' + ' <label>CODE1</label>' + ' </map>' + ' <map>' + ' <code>C0002</code>' + ' <label>CODE2</label>' + ' </map>' + ' </list>' + ' <map id="dataMap1">' + ' <lang>en</lang>' + ' <output>WebSquare</output>' + ' </map>' + ' <map id="undefined_data">' + ' <code>C0001</code>' + ' <label>CODE1</label>' + ' </map>' + '</map>'; // 예시 1-3) XML 유형 - 실행 결과 // XML 경로 "map/list/@id"에 정의된 "dataList1"와 XML 경로 "map/map[1]/@id"에 정의된 "dataMap1"은 화면에 정의된 데이터 객체와 ID가 동일하기 때문에 값이 할당됩니다. // 반면, XML 경로 "map/map[2]/@id"에 정의된 "undefined_data"는 화면에 정의된 데이터 객체의 ID가 일치하는 객체가 없기 때문에 무시됩니다. // 예시 2) 세 번째 인자(idArray) // 예시 2-1) ID 문자열 // 두 번째 인자 값의 데이터는 JSON 유형으로 정의되어 있으며, 총 2개의 KEY("dataList1", "undefined_data")로 구성되어 있습니다. $p.data.set( "JSON", { "dataList1": [ { "code": "C0001", "label": "CODE1" } ], "undefined_data": { "code": "C0001", "label": "CODE1" } }, ["dataList1", "dataMap1", "undefined_data", "undefined_id"] ); setBroadcast(flag)

모든 화면의 전체 DataCollection의 Broadcast 기능의 활성화 여부를 설정합니다.
첫 번재 인자(flag)를 false로 지정하면 Broadcast 기능이 비활성화됩니다.
Broadcast가 비활성화되면 데이터 객체들(DataMap, DataList 등)의 값 변경 시, 데이터 객체와 연결된 컴포넌트가 동기화되지 않습니다.
그 후, 첫 번재 인자(flag)를 true로 지정하여 Broadcast 기능을 활성화화면, 데이터 객체와 연결된 컴포넌트가 동기화됩니다.
(제약 사항)
이 함수를 호출한 페이지와 동일한 Window 객체에 생성된 DataCollection이 대상입니다.
IFrame의 소스 페이지 또는 '$p.openPopup'으로 생성한 팝업의 유형이 'iframePopup' 또는 'browserPopup'인 경우에는 Window 객체가 다르기 때문에 제외됩니다.
이 함수는 'Page Scope'이 적용되지 않습니다.
모든 화면에 영향을 미치므로 사용 시 주의가 필요합니다.

Parameter
  • Name
    flag
  • Type
    boolean
  • Required
    Y
  • Description

    [true, false] Broadcast 기능의 활성화 여부. 설정 값이 true이면 Broadcast가 활성화되어 DataCollection와 컴포넌트를 동기화합니다. 설정 값이 false이면 Broadcast가 비활성화되어 동기화되지 않습니다.

Sample

// 모든 화면의 모든 DataCollection의 Broadcast를 비활성화합니다. $p.data.setBroadcast(false); //데이터 객체 제어(삽입, 수정, 삭제) 로직 수행 // 모든 화면의 모든 DataCollection의 Broadcast를 활성화합니다. - 데이터 객체와 연결된 컴포넌트가 동기화됩니다. $p.data.setBroadcast(true);
