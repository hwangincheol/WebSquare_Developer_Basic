탭 컨트롤. 동적으로 탭을 생성/삭제. 최상위에 <div>가 위치하며 하위에 복합 태그들로 구성.

Property Detail

accessibility

웹 접근성 지원 기능을 사용할지에 대한 여부.
1. tab키의 포커스 순회 순서 변경 : 이전 컴포넌트 -> tablist -> content -> content 내부 순회 -> tab목록 버튼 -> 다음 컴포넌트 순서로 포커스가 이동한다. (shift tab키는 반대 방향)
2. tablist 영역을 순회하는 방법이 아래와 같이 변경됨
tablist에 포커스가 이동한 상태에서 좌우 방향키를 통해 tab을 이동한다.
좌우 방향키를 통해 활성화된 tab이 변경되면 그 즉시 content영역이 갱신된다.
tablist 이동에 순환 기능이 제공된다. 첫 번째 tab에서 왼쪽 방향키를 누를 경우 맨 마지막 tab으로 이동하며, 맨 마지막 tab에서 오른쪽 방향키를 누를 경우 첫 번째 tab으로 이동한다.
3. tabScroll="true"일 때 UI 변경 : 관련 UI들이 button으로 렌더링되고, tab 목록이 ul, li로 구성된다.
4. 아래의 속성값이 강제로 적용된다.

useAccessibilityMenu="true" useTabKeyOnly="false" tableRender="ul" tabScrollDiv="true" useBridge="true" useMoveNextTabFocus="false" changeActiveTab="true" useATagBtn="true" (옵션 설명) "false" (기본 값)웹 접근성 지원 기능을 사용하지 않음. "true" 웹 접근성 기능이 적용되어 키보드 기능 및 tabScroll관련 UI가 변경됨

true: TabControl 컴포넌트가 포함하는 모든 탭의 컴포넌트를 초기 화면 로딩 시 렌더링. (탭 및 컴포넌트 개수에 따라 브라우저의 렌더링 부하가 증가할 수 있음.)
