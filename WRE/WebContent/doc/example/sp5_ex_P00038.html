Submission의 ref와 target 속성 값에 따른 DataCollection 적용 예시

1개요

Submission의 ref속성과 target속성의 설정에 따른 전송 데이터와 응답 데이터의 DataCollection 할당 예시를 담고 있습니다.
설정에 따른 동작의 비교를 위해 만들어졌으며 구현 예시는 주요 설정 부분만 작성되어있습니다.

ref속성은 전송 데이터(request data)와 연관있고 target속성은 response data와 연관 있습니다.
target 속성값의 조건에 따라 서버의 응답 데이터(response data)가 dataCollection에 할당됩니다.

P00038

2구현된 기능

  • ref와 target을 설정하지 않은 예시

  • ref와 target 설정 예시- 1개의 data 객체 연결

  • ref와 target 설정 예시 - 2개의 data 객체 연결

  • singleMode 설정 예시

  • response 데이터와 dataCollection 자동 매칭 설정 예시

  • ref를 설정하고 requestData를 동적으로 할당한 예시

3예제 테스트 방법

１. 예시 유형 선택

[Submission의 ref와 target 설정 예시]에 있는 selectbox에서 실행할 예시를 선택합니다.

아래의 이미지와 같이 선택된 예시의 설명이 selectbox의 하위에 표현됩니다.

실행 할 Submission의 id, ref, target, action 속성값을 확인할 수 있습니다.

아래는 실행된 예제 화면의 일부 이미지입니다.

２. 실행하기

버튼 실행1 을 클릭합니다.

선택된 예시에 해당하는 submission이 실행됩니다.

３. 전송 데이터와 응답 데이터 확인하기

[Request, Response 정보] 하위의 textarea에 출력된 데이터를 확인합니다.

이 정보는 통신 후 Submission의 응답 객체에서 추출하여 출력되었습니다.

submission의 전역 ID와 서버로 전송한 request data, 서버에서 받은 response data가 출력됩니다.

4. DataCollection의 데이터 확인하기

응답 데이터(response data)가 DataCollection에 할당된 것을 확인합니다.

[이전/이후 DataCollection 정보]하위에 있는 textarea에 출력된 통신 이전/이후의 데이터를 비교합니다.

위에 있는 textarea에는 초기화된 통신 이전의 dataCollection 전체가 출력됩니다.

아래에 있는 textarea에는 통신 이후의 dataCollection 전체가 출력됩니다.

target에 설정된 정보에 따라 변경된 dataCollection의 데이터를 비교합니다.

4.6ref를 설정하고 requestData를 동적으로 할당한 예시

Submission을 호출하는 $p.executeSubmission API의 두번째 인자에 전송할 데이터를 JSON으로 할당합니다.
Submission에 설정된 ref의 값은 무시되고 API를 호출할 때 넘긴 request data가 할당됩니다.
//전송할 request data
var jsnRequestData = {
    "req_info" : {
        "userID" : "Dynamic_EDU003"
    }
};

//requestData를 동적으로 할당합니다. Submission에 설정한 ref는 무시됩니다.
$p.executeSubmission("sbm_json_dataListWithMessge_02", jsnRequestData);
