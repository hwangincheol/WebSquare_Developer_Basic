[AutoComplete] 입력창에서 Enter 또는 Tab 키가 입력되었을 때 이벤트 핸들러 등록하기

1개요

검색어 입력창에서 Enter 또는 Tab 키가 입력되었을 때(oneditenter)의 이벤트 핸들러를 등록하는 예제입니다.

P00067

2구현된 기능

  • 검색어 입력창에서 Enter 또는 Tab 키가 입력되었을 때 Key 정보 출력하기

3예제 테스트 방법

컴포넌트의 검색어 입력창에서 Enter 또는 Tab 키를 누릅니다.

그림 1.브라우저(Chrome) 실행 예시 - 키 입력

컴포넌트에서 포커스가 빠져나가고 로그 확인 영역에 이벤트 정보가 출력됩니다. (브라우저의 개발자 도구의 콘솔(console)탭을 통해 자세한 이벤트 정보를 확인 할 수 있습니다)

그림 2.로그 출력 영역 참고 이미지

4구현 예시

4.1속성 정의하기

속성 'nextTabByEnter'를 true로 정의합니다. 이 속성을 적용하지 않은 경우 기본값은 false입니다.

(속성 nextTabID가 지정되어 있는 경우, Enter 키 입력시 nextTabID 컴포넌트로 포커스를 이동 시킬지에 대한 여부를 정의합니다.)

그림 3.웹스퀘어5 SP5 스튜디오의 Property View(속성창) 예시

<!-- autoComplete의 소스 본문 예시 -->
<w2:autoComplete nextTabByEnter="true">
<!-- 중략 -->
</w2:autoComplete>

4.2이벤트 핸들러 정의하기

컴포넌트의 이벤트 oneditenter의 핸들러를 정의합니다.

예제 파일에서는 핸들러로 사용할 함수명을 scwin.acb_exam_oneditenter로 정의하였습니다.

그림 4.웹스퀘어5 SP5 스튜디오의 Property View(속성창)의 이벤트 탭 예시

스크립트 탭에서 핸들러 함수(scwin.acb_exam_oneditenter)를 정의합니다.
//컴포넌트 acb_exam의 이벤트 oneditenter 핸들러
scwin.acb_exam_oneditenter = function(e) {
    var tmpKeyCode;
    var tmpWhich;
    var tmpCode;
    var tmpKey;

    tmpKeyCode = e.keyCode;    // 13:Enter ,9:Tab
    tmpWhich = e.which;     // 13:Enter ,9:Tab
    tmpCode = e.code;
    tmpKey = e.key;

    //로직 작성
};
