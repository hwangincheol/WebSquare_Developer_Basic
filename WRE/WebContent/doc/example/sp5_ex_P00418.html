[PageList] 남은 페이지 표시 유형 설정하기

1개요

속성 'pagingType'의 설정 값 비교 예제입니다.
이 속성은 잔여 페이지(전체 페이지 % 화면에 표시할 페이지 수)를 화면에 표시하는 방식을 지정할 수 있습니다.
설정 값에 따른 동작은 다음과 같습니다.

  • "1" : [default] 속성 'pageSize'에 지정한 값을 기준으로 페이지를 표시합니다. 속성 'buttonShowType'의 설정 값이 '3'이면 이 설정은 지원하지 않습니다.

  • "2" : 남은 페이지만 표시합니다.

3예제 테스트 방법

3.1[공통 사항] 예제 구성

구성된 예제는 다음의 조건이 동일하게 설정되었습니다.

  • 화면에 표시할 페이지 수(속성 'pageSize') : '10'

  • 전체 페이지 수 : '14'

  • 선택된 페이지 : '12'

예제별 마지막 페이지를 화면에 표시(구성)하는 방식을 비교합니다.

3.2속성 pagingType의 설정 값을 "1"로 지정한 예시

STEP 1. 실행된 결과를 확인합니다.

영역 [(기본 설정) pagingType="1"]에 구성된 'PageList'를 확인합니다.
선택된 페이지는 '12'이며 화면에 표시된 페이지는 5 - 14입니다.
마지막 목록에 표시된 페이지가 5 - 14로 'PageList'에 설정 된 속성 'pageSize'(화면에 표시할 페이지 수)의 값 '10'에 맞춰 페이지 번호 10개가 표시됩니다.

그림 1.브라우저(Chrome) 실행 예시

3.3속성 pagingType의 설정 값을 "2"로 지정한 예시

STEP 1. 실행된 결과를 확인합니다.

영역 [pagingType="2"]에 구성된 'PageList'를 확인합니다.
선택된 페이지는 '12'이며 화면에 표시된 페이지는 11 - 14입니다.

그림 2.브라우저(Chrome) 실행 예시

4구현 예시

4.1속성 pagingType 설정하기

속성을 정의합니다.

  • [필수] pagingType="옵션 값"

    (옵션 값)

    - "1" : [default] 속성 'pageSize'에 지정한 값을 기준으로 페이지를 표시합니다. 속성 'buttonShowType'의 설정 값이 '3'이면 이 설정은 지원하지 않습니다.

    - "2" : 남은 페이지만 표시합니다.

  • [선택] pageSize="화면에 표시할 페이지 번호의 개수"

    기본 설정 값은 "10"입니다.
